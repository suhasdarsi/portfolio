// PRIORITY ISSUES
P0

Mobile search trigger has no visible label

On viewports < 640px, the search button shows only a 12×12px magnifying glass icon. The aria-label is present for screen readers, but sighted users on mobile see an unlabeled icon with no textual context.

FIX: /design interaction on the search trigger to add a visible label or meaningful icon sizing on all viewport sizes.
P1

Prose may overflow on very narrow viewports

.prose-custom sets font-size: 1.375rem (22px) with max-width: 75ch. At 22px per character, 75ch ≈ 1650px — the overflow-wrap: break-word prevents hard breakage, but on a 320px phone, long unbroken strings, wide tables, or oversize images may force horizontal scroll.

FIX: /design responsive on .prose-custom to clamp the font size on narrow viewports and add safe overflow handling for embedded content.
P2

Orphan accent colors defined but unused

--color-signal-blue: #517cd9 and --color-ember: #cd5f37 are defined in the theme but never referenced in any component or page. Not harmful but signals unfinished color-system work.

FIX: Either purge them or use intentionally (e.g., ember for underline accents, signal-blue for external link indicators).
P3

Fragile CSS in prose-custom

All heading rules in .prose-custom use !important on every property — 21 declarations total. This creates a specificity trap where any override must also use !important. Fragile and hard to maintain.

FIX: /design tokenize to restructure prose-custom with higher-specificity selectors instead of !important.
